Agora que você domina a estrutura básica de um **Ativador + Efeito**, é hora de adicionar **inteligência** aos seus sistemas. As Condições permitem que você valide requisitos antes de executar um Efeito, criando comportamentos condicionais ao invés de respostas automáticas.

## O que uma Condição faz?

Uma **Condição** funciona como um "vigia lógico": quando acionada, ela verifica se um requisito foi atendido. Se sim, a execução continua normalmente. Se não, o sistema é abortado e o Efeito não é executado.

<Aside type="tip" title="Dica de Ouro">
Em um sistema com múltiplas Condições, **TODAS** elas precisam ser verdadeiras por padrão (lógica "E"). Se qualquer uma for falsa, toda pilha não é executada. Mais à frente você aprenderá como usar Extras para criar lógica "OU".
</Aside>

## Por que usar Condições?

Imagine um sistema de **teleporte exclusivo para membros de um grupo**. Sem uma Condição, qualquer pessoa que pisasse no ativador seria teletransportada. Com uma Condição, apenas membros do grupo teriam acesso.

Casos de uso comuns:
- **Controle de acesso** (grupo, equipe, permissões)
- **Validações de estado** (mobis em posições específicas)
- **Filtros de tempo** (dia/noite, horários)
- **Proteção de sistemas** (evitar execuções indevidas)

## Estrutura: Ativador + Condição + Efeito

<Steps>

1. **ATIVADOR:** Detecta o evento (ex: usuário pisa em um mobi)

2. **CONDIÇÃO:** Valida se o usuário atende ao requisito (ex: "É membro do grupo X?")

3. **EFEITO:** Executa apenas se a resposta for "sim" (ex: teleporta para o prêmio)

</Steps>

Se a Condição retornar "não", o fluxo é interrompido imediatamente.

## Exemplo Prático: Acesso Exclusivo

Você quer criar um **sala VIP** acessível apenas para membros de um grupo específico:

### Configuração:

<Steps>
1. **Ativador:** Habbo anda no Mobi
    1. Selecione o piso de entrada
2. **Condição:** Membro do Grupo.
    1. Selecione o grupo permitido
3. **Efeito:** Teleportar para Mobi.
    1. Selecione o destino da área VIP
</Steps>

### O que acontece:
- Um usuário **membro do grupo** pisa no piso → passa pela Condição → é teleportado
- Um usuário **não membro** pisa no piso → falha na Condição → nada acontece

## Combinando múltiplas Condições

Às vezes você precisa de **mais de uma validação**. Por exemplo:
- Usuário deve estar em uma **equipe específica** E
- Estar em um **grupo específico** E
- Ter uma **pontuação mínima**

Neste caso, adicione todas as 3 Condições à mesma pilha. Se qualquer uma falhar, a execução é abortada.

<Aside type="caution" title="Atenção">
Se você quer usar lógica "OU" (Condição A **OU** Condição B), você precisa de um **Extra** especial. Siga para a próxima sessão para aprender sobre Seletores e Extras.
</Aside>

## Condições Negativas

Algumas Condições possuem versões **negativas** (com cor preta), que realizam a verificação inversa:

- **"Membro do Grupo"**
- **"NÃO é Membro do Grupo"**

Use-as quando precisar rejeitar algo ao invés de aceitar.
