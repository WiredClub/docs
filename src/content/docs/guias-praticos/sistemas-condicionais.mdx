Agora que você domina a estrutura básica de um **Ativador + Efeito**, é hora de adicionar **inteligência** aos seus sistemas. Nesta seção, você aprenderá sobre as **Condições**, que funcionam como "filtros" para validar requisitos antes de executar um Efeito. Com elas, seus sistemas deixam de ser respostas automáticas e passam a ter comportamentos condicionais, permitindo criar mecânicas mais complexas e personalizadas.

## O que uma Condição faz?

Uma **Condição** funciona como um "vigia lógico": quando acionada, ela verifica se um requisito foi atendido. Se sim, a execução continua normalmente. Se não, o sistema é abortado e o Efeito não é executado.

## Exemplo Prático: Acesso Exclusivo

Imagine que você quer criar uma **sala VIP** acessível apenas para membros de um grupo específico. Sem uma Condição, qualquer pessoa que pisasse no ativador seria teletransportada para a sala VIP. Com uma Condição, apenas membros do grupo teriam acesso. Podemos estruturar isso da seguinte forma:

<Tabs>
  <TabItem label="Configuração">
    <Steps>
1. **Ativador:** Habbo anda no Mobi
    1. Selecione o piso de entrada
2. **Condição:** Membro do Grupo.
    1. Selecione o grupo permitido
3. **Efeito:** Teleportar para Mobi.
    1. Selecione o destino da área VIP
</Steps>
  </TabItem>
  <TabItem label="PseudoWired">
```yaml
QUANDO
  Habbo anda no mobi: Piso Escuro

SE
  Membro do Grupo: "VIP"

ENTÃO
  Teleportar para Mobi: Área VIP
```
</TabItem>
</Tabs>

### O que acontece:
- Um usuário **membro do grupo** pisa no piso → passa pela Condição → é teleportado
- Um usuário **não membro** pisa no piso → falha na Condição → nada acontece
