Esta condição do sistema Wired verifica se **todos os mobis selecionados possuem pelo menos um usuário sobre eles** no momento da execução. Seu funcionamento consiste na seleção de um ou mais itens que serão monitorados: quando o ativador associado for disparado, por padrão a condição apenas permitirá a execução dos efeitos caso cada um desses mobis esteja ocupado por um avatar. Nessa configuração, caso qualquer um dos itens esteja vazio, a condição falha e interrompe a execução do restante da pilha.

<Infobox />

## Exemplos de uso
- **Portas que exigem presença**: Conectando esta condição a um efeito que abre uma porta, é possível criar entradas que só se abrem quando todos os pontos de ativação estão ocupados, ideal para jogos de equipe ou desafios de cooperação.
- **Jogos de coordenação**: Em jogos onde os usuários precisam se posicionar em pontos específicos para avançar, a condição pode ser usada para validar que todos os participantes estão nos locais corretos antes de permitir a progressão, incentivando a comunicação e o trabalho em equipe.

## Versão negativa
A versão negativa desta condição, chamada "Mobis NÃO têm usuários em cima", inverte a lógica, por padrão permitindo a execução dos efeitos apenas quando **nenhum dos mobis selecionados tiver um usuário sobre eles**. Essa variação é útil para criar desafios onde a ausência de presença é o requisito, como para criar armadilhas que se ativam quando os usuários deixam um local específico.
