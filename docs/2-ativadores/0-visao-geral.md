# O que é um Ativador?
O Ativador é o componente que "escuta" eventos específicos no quarto para iniciar uma execução. Ele é o "Quando" da automação e funciona como uma bateria que fornece o impulso inicial para a pilha.

*   **Identificação Visual:** Todos os ativadores são representados por caixas na cor **marrom**.[3]
*   **Regra de Ouro:** Um ativador sozinho não faz nada; ele precisa obrigatoriamente estar empilhado com pelo menos um **Efeito** para que algo aconteça no quarto.[2, 4]

## Como configurar
Descreva o processo básico de interface:
*   **Programação:** Para configurar um Ativador, o usuário deve clicar duas vezes (double-click) sobre o mobi.[2, 4]
*   **Seleção de Alvos:** Dependendo do tipo, você precisará selecionar quais mobis ou usuários dispararão o gatilho, ou quais palavras-chave serão monitoradas.[4, 5]

## Categorias de Ativadores
Agrupe os ativadores por comportamento para facilitar o aprendizado:
*   **Baseados em Usuários:** Gatilhos disparados por ações diretas de Habbos, como entrar no quarto, andar sobre um mobi, clicar em um objeto ou dizer uma palavra-chave.[1]
*   **Baseados em Tempo (Repetição):** Ativadores que funcionam em ciclos automáticos, como o "Repetir Efeito" (ciclos curtos) ou "Periodicamente Longo" (ciclos de até 10 minutos).[3, 1]
*   **Baseados em Sistemas e Jogos:** Gatilhos vinculados ao estado de um jogo, como o início ou fim de uma partida, ou quando uma equipe atinge uma pontuação específica.[1]
*   **Baseados em Bots e Pets:** Ativadores que detectam quando um bot alcança um usuário ou um mobi específico.[3, 1]

## Entendendo a "Fonte de Gatilho"
Introduza o conceito técnico de **Trigger Source (AQ)**. Explique que o ativador identifica *quem* iniciou a ação (se foi um humano, bot ou animal) e passa essa informação para o resto da pilha.[6] Isso é crucial para que efeitos como "Teleportar para Mobi" saibam exatamente quem devem mover.[2]

### 5. Exemplo Prático de Fluxo
Apresente uma combinação simples para ilustrar a lógica:
*   **Ativador:** Habbo anda no Mobi.
*   **Efeito:** Teleportar para Mobi.
*   **Resultado:** Assim que o usuário pisa no item X, ele é enviado instantaneamente para o item Y.[1, 5]
